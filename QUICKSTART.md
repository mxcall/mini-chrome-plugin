# 快速开始指南

## 第一步:启动后端服务

### 方式一:使用启动脚本(推荐)
双击运行 `start_server.bat` 文件,脚本会自动:
- 检查Python环境
- 安装必要的依赖
- 创建uploads目录
- 启动后端服务

### 方式二:手动启动
1. 打开命令行窗口
2. 进入项目目录
3. 安装依赖:
   ```
   pip install flask flask-cors requests
   ```
4. 运行服务:
   ```
   python main.py
   ```

启动成功后,你会看到类似信息:
```
文件上传服务启动中...
监听地址: localhost:19666
文件保存目录: C:\...\uploads
按 Ctrl+C 停止服务
```

## 第二步:安装Chrome插件

1. 打开Chrome浏览器
2. 地址栏输入: `chrome://extensions/`
3. 打开右上角的"开发者模式"开关
4. 点击"加载已解压的扩展程序"
5. 选择项目中的 `chrome-extension` 文件夹
6. 插件安装成功!

## 第三步:测试文件上传

### 方法一:使用Chrome插件测试
1. 点击浏览器工具栏的插件图标
2. 点击"选择文件"按钮
3. 选择要上传的文件
4. 点击"上传文件"按钮
5. 等待上传完成,查看结果提示

### 方法二:使用测试脚本验证后端
1. 确保后端服务已启动
2. 打开新的命令行窗口
3. 运行测试脚本:
   ```
   python test_backend.py
   ```
4. 按照提示操作,查看测试结果

## 验证上传结果

上传的文件会保存在项目根目录下的 `uploads` 文件夹中:
- 如果文件名不冲突,使用原始文件名
- 如果文件名冲突,会自动添加时间戳重命名

例如:
- 原始文件: `document.pdf`
- 重复上传后: `document_20251107153025.pdf`

## 常见问题

**Q: 插件提示"上传失败,请确保后端服务正在运行"**
- 检查后端服务是否启动
- 确认端口19666未被占用
- 查看后端服务终端是否有错误信息

**Q: 上传大文件时间很长?**
- 这是正常的,2GB文件需要一定时间
- 请耐心等待,不要关闭浏览器
- 后端会流式处理,不会占用过多内存

**Q: 找不到上传的文件?**
- 检查后端服务启动时显示的保存目录
- 默认在项目根目录的 `uploads` 文件夹中
- 注意可能有时间戳重命名

**Q: Chrome插件无法加载?**
- 确认开发者模式已开启
- 检查manifest.json文件是否正确
- 查看扩展程序页面是否有错误提示

```
├── chrome-extension/     # Chrome插件文件夹
│   ├── manifest.json    # 插件配置文件
│   ├── popup.html       # 插件弹出界面
│   ├── popup.js         # 插件交互逻辑
│   ├── popup.css        # 插件样式文件
│   └── icon.png         # 插件图标
├── main.py              # Python后端服务
├── start_server.bat     # Windows启动脚本
├── test_backend.py      # 后端测试脚本
├── pyproject.toml       # Python项目配置
└── README.md            # 项目说明文档
```

## 停止服务

在后端服务运行的命令行窗口按 `Ctrl+C` 即可停止服务。

## 技术支持

遇到问题?请检查:
1. Python版本 >= 3.8.20
2. Chrome版本 >= 104
3. 所有依赖已正确安装
4. 端口19666未被占用
5. 防火墙允许本地连接
